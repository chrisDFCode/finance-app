## 🎯 What Each File Does

### Core Application
- **src/App.jsx**: Root React component that sets up the app structure
- **src/main.jsx**: Entry point that mounts the app to DOM
- **src/pages/Dashboard.jsx**: Main page orchestrating all features

### Components (User Interface)
- **AddExpenseForm.jsx**: Form for adding new expenses with validation
- **ExpenseTable.jsx**: Table displaying all expenses with delete buttons
- **CategoryFilter.jsx**: Buttons to filter expenses by category
- **MonthlySummary.jsx**: Shows monthly spending summary with breakdown
- **ChartComponent.jsx**: Bar chart & doughnut chart visualizations
- **StatsSection.jsx**: Displays total, average, and transaction count

### Backend Integration
- **supabaseClient.js**: Handles all database operations (CRUD)
- **useExpenses.js**: Custom hook for managing expense state
- **helpers.js**: Utility functions for formatting and calculations

### Styling
- **global.css**: CSS variables, base styles, form components
- **dashboard.css**: Layout structure and dashboard styling
- **components.css**: Component-specific styles with responsive design

### Configuration
- **package.json**: Defines all dependencies and npm scripts
- **vite.config.js**: Vite build tool configuration
- **vercel.json**: Vercel deployment configuration
- **.env.example**: Template for environment variables
- **index.html**: HTML root element and meta tags

### Database
- **database.sql**: PostgreSQL schema with table, indexes, and functions

### Docker
- **Dockerfile**: Multi-stage build for production image
- **docker-compose.yml**: Service composition for local/prod
- **.dockerignore**: Files to exclude from Docker build

### Setup & Automation
- **setup.sh**: Automated setup for Linux/Mac
- **setup.bat**: Automated setup for Windows

### Documentation
- **INDEX.md**: Navigation guide to all documentation
- **README.md**: Quick project overview
- **SETUP_GUIDE.md**: Complete setup instructions (11 sections)
- **IMPLEMENTATION_GUIDE.md**: Technical architecture explanation
- **API_DOCUMENTATION.md**: Function and component reference
- **QUICK_REFERENCE.md**: Cheatsheet for commands and quick lookup
- **DEPLOYMENT_CHECKLIST.md**: Pre-launch verification checklist
- **DOCKER_GUIDE.md**: Docker build and deployment guide
- **PROJECT_SUMMARY.md**: Complete project overview

## 💾 Database Schema

### Single Table: `expenses`
```sql
Columns:
- id (BIGSERIAL PRIMARY KEY)
- amount (DECIMAL 0.01-99999999.99)
- category (VARCHAR 50)
- date (DATE)
- note (TEXT, optional)
- created_at (TIMESTAMP AUTO)
- updated_at (TIMESTAMP AUTO)

Indexes:
- idx_expenses_date
- idx_expenses_category
- idx_expenses_created_at
```

## 🔧 Scripts Available

### Development
```bash
npm run dev          # Start development server (port 5173)
npm install          # Install dependencies
npm cache clean      # Clear npm cache
```

### Production
```bash
npm run build        # Create production build
npm run preview      # Preview production build
```

### Docker
```bash
docker build -t finance-app .     # Build image
docker run -p 3000:3000 finance-app  # Run container
docker-compose up                 # Run with compose
```

### Deployment
```bash
vercel              # Deploy to Vercel (staging)
vercel --prod       # Deploy to Vercel (production)
```

### Setup
```bash
./setup.sh         # Linux/Mac automated setup
setup.bat          # Windows automated setup
```
